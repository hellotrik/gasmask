# EnvMask 使用说明（极简版）

> 目标：像 Gas Mask 管 hosts 一样管环境变量。\n+> 你只需要会 3 个动作：**开关配置包**、**加临时变量**、**让终端自动生效**。\n+
## 1) 开关配置包（Profiles）

- 菜单栏 `Env Mask -> Profiles(配置包开关)`\n+  - 勾选 `[x]`：启用该配置包\n+  - 取消：禁用\n+
**说明**：EnvMask 支持同时启用多个 Profiles，最终环境变量会按 `priority` 合并（高优先级覆盖低优先级）。\n+
## 2) 临时变量（Temp）

- 菜单栏 `Env Mask -> Temp(临时变量) -> 添加变量...`\n+  - 覆盖：`KEY=VALUE`\n+  - 追加：`KEY+=VALUE`（适合 `PATH`）\n+  - 删除：`unset KEY`\n+- `清空 Temp`：一键删除所有临时变量，并关闭 Temp\n+
**建议用法**：临时变量用完就清空，不污染长期配置。\n+
## 3) 让终端自动生效（只需一次）

- 菜单栏顶部会显示：`终端自动生效: 已开启/未开启`\n+- 点击：\n+  - `开启终端自动生效(写入 ~/.zshrc)`\n+  - 或 `关闭终端自动生效(从 ~/.zshrc 移除)`\n+
**生效范围**：\n+- 开启后：**新开的终端会话**会自动带上当前 Profiles+Temp 的环境变量\n+- 已打开的终端：需要新开窗口，或手动执行菜单里的“复制终端手动生效命令”\n+
## 打开编辑器有什么用？\n+\n+- 菜单栏 `Env Mask -> 打开编辑器...`\n+- 用于：\n+  - 新建/删除 Profile（像文件一样管理）\n+  - 给 Profile 增删变量\n+  - 调整 `priority`\n+  - 查看最终 Resolved 结果\n+
